# Contributing

## Branch Strategy

Never commit directly to main.

Use:

feature/*

↓

develop

↓

release/*

↓

main

---

## Commit Convention

feat(module): new feature

fix(module): bug fix

docs(module): documentation update

refactor(module): code cleanup

---

## Pull Request Rules

* Build must pass.
* Documentation must be updated.
* Architecture rules must not be violated.
* New modules require documentation.
